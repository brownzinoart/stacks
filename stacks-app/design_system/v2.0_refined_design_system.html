‚ú® V2.0 - REFINED BRUTALISM

Inspired by MotherDuck - softer, smarter, still bold

What Changed?

Version 2.0 refines the brutalist aesthetic with warm neutrals, contextual border weights, and generous spacing ‚Äî perfect for a book-focused reading app.

üé® Key Improvements

  • Warm beige background (#F4EFEA) - easier on eyes for long reading sessions
  • Contextual borders - 5px for cards/CTAs, 3px for badges, 2px for inputs
  • Offset shadows (-8px 8px style) - playful MotherDuck-inspired dimensionality
  • Generous spacing (36-60px padding) - breathing room between elements
  • Softer yellow accent (#EAC435) - golden tone instead of bright neon
  • Refined color palette - cyan, teal, coral accents on warm neutral base

Migration Guide: v1.0 ‚Üí v2.0

Quick reference for updating from v1.0 to v2.0

// Color Token Changes: --bg-primary: #fafafa ‚Üí #F4EFEA (warm beige) --accent-yellow: #FFDE00 ‚Üí #EAC435 (softer golden) // Border Weight Variables (NEW): --border-heavy: 5px (cards, CTAs) --border-medium: 3px (badges, secondary) --border-light: 2px (inputs, dividers) // Shadow Direction Changed: Old: 6px 6px 0px (bottom-right) New: -8px 8px 0px (offset left-down for playful feel) // Spacing Scale Updated: --space-lg: 24px ‚Üí 36px --space-xl: 32px ‚Üí 48px --space-2xl: 40px ‚Üí 60px

  • ‚Ä¢ Background color changed (update contrast checks)
  • ‚Ä¢ Shadow direction reversed (update hover states)
  • ‚Ä¢ Spacing increased (check layouts for overflow)
  • ‚Ä¢ Border weights now contextual (use variables instead of fixed values)

Template System

Three-level template hierarchy following atomic design methodology

  • Level 1: Layout Templates - Base structural patterns (Centered Feed, Tabbed Hub, Search+Content, Modal/Overlay)
  • Level 2: Page Templates - Complete page implementations (Home Feed, Profile Hub, Discovery, etc.)
  • Level 3: Variants - Context-specific modifications (Empty States, Loading States, Error States)

Layout Templates (Level 1)

Base structural patterns that can be reused across multiple pages

Structure: Vertical scroll, single column, centered max-width container
Spacing: space-y-8 between cards
Used by: Home Feed, Settings

Tabbed Hub Layout

Scroll: Horizontal (overflow-x-auto)

Search + Content Layout

Layout: Mix of horizontal scroll + grid

Modal/Overlay Layout

Sizes: full, large, medium, small
Used by: Book details, stack creation, onboarding
Best for: Focused interactions

Usage Guidelines

  • ‚Ä¢ Always provide context: Explain why the state is empty and what the user can do
  • ‚Ä¢ Use appropriate variants: Match gradient to page theme (primary for general, accent for search, etc.)
  • ‚Ä¢ Include actions when possible: Give users a clear next step
  • ‚Ä¢ Size appropriately: Small for inline sections, large for full-page empty states
  • ‚Ä¢ Choose meaningful icons: Use icons that clearly represent the missing content
